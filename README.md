## 项目背景

本项目是**飞书AI大赛企业命题赛道**参赛作品，命题企业为**益和宠品**。

益和宠品作为宠物食品行业的领军品牌，面临着新品研发周期长、宠物适口性测试成本高、消费者需求快速变化等挑战。宠物食品行业的特殊问题在于：购买者是宠物主人，而使用者是宠物。如何在新品正式上市之前，准确预测宠物是否愿意吃、主人是否愿意买，降低新品失败风险，提升爆品率是企业最核心的业务问题。

***

## 企业命题分析

### 命题解读

益和宠品作为宠物零食行业的领军品牌，面临三大核心挑战：

1. **口味与功能趋势难捕捉**：高频创新节奏下，消费者需求快速变化
2. **Z世代宠主深度追问**：关注配料来源、钙磷比、添加剂，追求"保健品级"功效（美毛、护肠、洁齿）
3. **配方研发与功效验证周期长**：传统活体试食试验周期长达1.5-2年，适口性、安全性快速验证仍是瓶颈

**命题核心问题**：如何借助AI打造下一代宠物健康零食爆款？

### 评审维度

- **对企业业务问题的理解**：是否准确把握宠物食品行业三大核心挑战（口味趋势、功能需求、功效验证）
- **AI解决方案设计能力**：方案的完整性、创新性和技术可行性
- **技术架构完整性**：系统设计的合理性、可扩展性和落地性
- **创新性和落地价值**：是否具有独特的创新点和明确的商业价值

## 核心痛点

### 行业痛点

1. **口味与功能趋势难捕捉**：高频创新节奏下，消费者需求快速变化，传统调研方法滞后
2. **Z世代宠主深度追问**：关注配料来源、钙磷比、添加剂，追求"保健品级"功效（美毛、护肠、洁齿），关注点随社交媒体情绪流向变化极快
3. **配方研发与功效验证周期长**：功能性零食研发从配方设计到多轮宠物试食验证，传统活体试验周期长达1.5-2年，适口性、安全性快速验证仍是瓶颈
4. **消费者不是最终使用者**：购买决策由宠主完成，但实际使用者是宠物，两者需求存在差异
5. **产品-市场匹配困难**：产品团队难以精准把握宠主对新品设计的真实反馈

### 企业痛点

> **"这个产品做出来，宠物愿不愿吃？主人愿不愿买？功效能否快速验证？"**

这是益和宠品产品团队在新品研发过程中最常问的问题，也是最核心的业务痛点。

## 项目介绍

### 项目名称

**PetMind —— AI宠物零食共创实验室**

### 项目定位

基于飞书AI生态的虚拟宠物适口性测试与虚拟宠主消费者测试智能体系统。

### 核心理念

帮助宠物食品企业在新品正式上市之前，通过AI模拟宠物适口性和目标宠主进行虚拟市场验证。

### 核心流程

```
用户反馈 + 配方数据 + 宠物营养知识
        ↓
  Insight Agent 生成消费者洞察
        ↓
  构建虚拟宠主模型（画像 + 决策逻辑）
        ↓
  产品团队输入新品概念
        ↓
  AI宠物适口性预测 + 虚拟宠主测试反馈
        ↓
  Product Agent 优化产品方案
        ↓
  Marketing Agent 生成营销内容
        ↓
  Growth Agent 提供迭代建议
```

## 核心创新

### 创新1：AI虚拟适口性预测

宠物食品最大的特殊问题：消费者不是最终使用者。

- **购买者**：宠物主人
- **使用者**：宠物

传统流程：研发配方 → 制作样品 → 宠物测试 → 调整（周期长、成本高）

我们的方案：建立"宠物食品知识库 + 历史配方数据 + 用户反馈数据"，利用AI建立预测模型。

输入：原料组合、营养成分（钙磷比等）、产品定位、历史反馈

输出：适口性风险预测、可能问题、优化建议

> **AI不是替代真实宠物实验，而是在进入真实测试前完成低成本筛选，提高研发效率。**

### 创新2：AI功效验证加速

功能性零食的研发周期长达1.5-2年，核心瓶颈在于功效验证。

传统流程：配方设计 → 多轮活体试食 → 功效评估 → 调整

我们的方案：

- **营养成分分析**：基于宠物营养学知识，评估配方的钙磷比、蛋白质含量等关键指标
- **功效预测模型**：基于历史数据和营养知识，预测功能性成分的效果（美毛、护肠、洁齿）
- **风险预警**：识别潜在的营养风险和安全隐患

### 创新3：AI虚拟宠主消费者测试

宠物食品购买决策由主人完成。Insight Agent根据电商评论、社交媒体讨论、用户画像数据，生成不同类型虚拟消费者。

例如：

- **Z世代成分党**：22-28岁女性，关注配料来源、钙磷比、添加剂、保健品级功效，决策逻辑：成分透明度 > 功能性 > 品牌 > 价格
- **科学养宠家长**：关注营养配比、钙磷比、功能性（美毛/护肠/洁齿），追求科学喂养
- **多宠家庭主理人**：关注性价比、宠物健康、产品复购

新品概念输入后，AI模拟不同宠主反馈：购买意愿、担忧因素、产品改进建议。

## Agent架构

### 1. Insight Agent（核心）

**定位**：AI市场研究专家 + 虚拟宠主生成器 + 功效验证分析师

**功能**：

- 分析用户评论，提炼消费痛点（口味趋势、功能需求、成分关注）
- 分析宠物食品趋势（美毛、护肠、洁齿等功能性需求）
- 生成虚拟宠主画像（如Z世代成分党、科学养宠家长、多宠家庭主理人等）
- 进行新品概念测试（模拟宠主购买决策）
- 生成宠物适口性预测（基于配方数据和历史反馈）
- 进行营养成分分析（钙磷比、蛋白质含量等关键指标评估）

**输入**：品牌信息、产品品类、用户评论数据、行业报告、配方数据、宠物营养知识

**输出**：用户画像、消费趋势、购买因素、测试反馈、适口性预测、功效预测

### 2. Product Agent

**定位**：宠物食品研发助手

**功能**：

- 根据用户需求、适口性预测、功效验证结果优化配方方向
- 输出产品定位和核心卖点（突出功能性、成分透明度）
- 提出产品迭代建议（针对钙磷比、添加剂、功能性成分）

**输入**：新品概念、虚拟宠主测试反馈、适口性预测、功效预测、消费者洞察

**输出**：优化后的产品方案、配方优化建议（钙磷比调整、功能性增强）、产品定位报告、核心卖点清单

### 3. Marketing Agent

**定位**：宠物品牌内容增长助手

**功能**：

- 根据产品特点生成小红书种草内容、抖音短视频脚本、电商详情页、广告文案
- 突出产品的功能性卖点（美毛、护肠、洁齿）和成分透明度

**输入**：产品方案、核心卖点（含功效信息）、目标人群

**输出**：多平台营销内容、传播策略

### 4. Growth Agent

**定位**：AI增长分析师

**功能**：

- 分析销售数据、用户反馈、内容表现
- 提供迭代优化建议、预测市场表现
- 识别功能性零食市场机会

**输入**：虚拟测试结果、营销效果数据、行业趋势、功效验证结果

**输出**：增长分析报告、产品迭代建议、市场机会分析（功能性零食方向）

## 技术架构

### 飞书AI生态

| 飞书能力      | 应用场景                             |
| --------- | -------------------------------- |
| 飞书群聊/智能入口 | 用户交互界面，产品团队发起虚拟测试                |
| 飞书自定义智能体  | 部署四个核心Agent，实现自然语言交互             |
| 飞书多维表格AI  | 数据管理和分析，存储消费者洞察、测试结果、配方数据        |
| 飞书知识库     | RAG知识检索，存储宠物营养知识、品牌资料、产品数据库、行业报告 |
| 飞书工作流     | 编排Agent协作流程，实现自动化测试和迭代           |

### 技术能力

- **大语言模型**：驱动Agent的核心推理能力
- **RAG知识增强**：基于知识库提供精准的消费者洞察、适口性预测和功效验证
- **Multi-Agent协作**：多个Agent按业务流程协作完成任务
- **虚拟消费者建模**：从真实数据中构建宠主决策模型（关注成分、钙磷比、功能性）
- **适口性预测模型**：基于历史配方数据和反馈构建预测模型
- **功效验证模型**：基于宠物营养学知识，评估钙磷比、功能性成分效果

### 早期验证方案

- **Dify Workflow**：用于早期逻辑验证、流程调试
