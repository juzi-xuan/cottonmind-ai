### 1.2 架构设计原则

1. **核心引擎优先**：虚拟消费者引擎是整个系统的核心，所有Agent围绕其展开工作

2. **飞书生态原生**：系统设计充分考虑飞书AI生态的能力边界，最大化利用飞书原生能力

3. **模块化设计**：每个Agent独立封装，通过标准化接口协作

4. **数据驱动**：所有决策基于真实数据，确保虚拟测试的可信度

5. **迭代闭环**：支持产品-测试-优化的持续迭代循环

## 2. Agent设计

### 2.1 Insight Agent（核心）

#### 身份

**AI消费者研究专家 + 虚拟消费者生成器**

#### 核心职责

作为系统的核心引擎，负责：
- 从真实数据中提炼消费者洞察
- 构建具有决策逻辑的虚拟消费者模型
- 模拟消费者对新品概念的反馈

#### 输入

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 品牌信息 | 文本 | 飞书知识库 | 品牌定位、品牌调性、产品矩阵 |
| 产品品类 | 文本 | 用户输入 | 如：婴儿湿巾、棉柔巾、浴巾等 |
| 用户评论数据 | 结构化数据 | 数据存储层 | 电商平台、社交媒体评论 |
| 行业报告 | 文档 | 飞书知识库 | 行业趋势、市场规模、竞品分析 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 消费者洞察报告 | 结构化报告 | 痛点、需求、偏好分析 |
| 虚拟消费者模型 | JSON对象 | 包含画像特征和决策逻辑 |
| 测试反馈结果 | 结构化数据 | 不同消费者群体的反馈评分 |

#### 工具

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| RAG知识检索 | 从知识库中检索相关信息 | 分析用户评论、行业报告 |
| 数据统计分析 | 对评论数据进行统计分析 | 提炼消费痛点和趋势 |
| 画像聚类 | 对消费者进行群体聚类 | 生成消费者细分群体 |
| 决策树生成 | 生成消费者购买决策逻辑 | 构建虚拟消费者决策模型 |

#### 知识库

| 知识库 | 内容 | 更新频率 |
|--------|------|----------|
| 用户评论库 | 电商平台、社交媒体用户评论 | 实时 |
| 行业报告库 | 行业分析报告、市场研究报告 | 月度 |
| 品牌知识库 | 品牌历史、产品信息、品牌调性 | 季度 |
| 消费者画像库 | 消费者细分、人群特征 | 季度 |

#### Prompt设计

**角色设定**：
```
你是一名资深消费者研究专家，专注于母婴棉品行业。
你拥有丰富的市场研究经验，擅长从海量数据中提炼洞察，并能够构建逼真的虚拟消费者模型。
你的目标是帮助产品团队在新品上市前预测市场反应。
```

**核心指令**：
```
基于以下数据，完成消费者洞察分析并生成虚拟消费者模型：

1. 用户评论数据：{{reviews_data}}
2. 行业报告：{{industry_reports}}
3. 品牌信息：{{brand_info}}

请输出：
1. 消费者洞察报告：
   - 核心消费痛点（TOP 10）
   - 消费者需求趋势
   - 价格敏感度分析
   - 购买决策因素排序

2. 虚拟消费者模型（至少5个细分群体）：
   - 群体名称
   - 核心特征（年龄、地域、消费能力、育儿阶段等）
   - 购买决策逻辑（决策树）
   - 产品偏好权重

3. 测试反馈模拟：
   - 针对新品概念{{product_concept}}，模拟各群体的反馈
   - 购买意愿评分（1-10分）
   - 主要顾虑点
   - 改进建议
```

### 2.2 Product Agent

#### 身份

**AI产品经理**

#### 核心职责

根据虚拟消费者的测试反馈，优化产品方案，提炼核心卖点。

#### 输入

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 新品概念 | 文本 | 用户输入 | 产品初步设计方案 |
| 虚拟测试反馈 | 结构化数据 | Insight Agent | 各消费者群体的反馈 |
| 消费者洞察 | 报告 | Insight Agent | 消费痛点、需求趋势 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 优化后产品方案 | 结构化文档 | 产品定位、功能设计、定价策略 |
| 产品定位报告 | 文本报告 | 目标人群、差异化定位、竞争优势 |
| 核心卖点清单 | 列表 | 产品核心价值主张 |

#### 工具

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| 竞品分析 | 分析竞品产品特征 | 制定差异化策略 |
| 定价策略分析 | 分析价格区间和敏感度 | 制定定价方案 |
| 卖点提炼 | 从产品特征中提炼核心卖点 | 生成营销素材 |

#### 知识库

| 知识库 | 内容 | 更新频率 |
|--------|------|----------|
| 竞品数据库 | 竞品产品信息、定价、评价 | 月度 |
| 产品知识库 | 品牌产品历史、技术参数 | 季度 |
| 定价策略库 | 行业定价策略、价格区间 | 季度 |

#### Prompt设计

**角色设定**：
```
你是一名资深产品经理，专注于母婴棉品行业。
你擅长根据消费者反馈优化产品方案，制定产品定位和核心卖点。
你能够平衡用户需求、技术可行性和商业价值。
```

**核心指令**：
```
基于以下信息，优化产品方案并输出产品定位和核心卖点：

1. 原始新品概念：{{product_concept}}
2. 虚拟消费者测试反馈：{{test_feedback}}
3. 消费者洞察报告：{{insight_report}}

请输出：
1. 优化后的产品方案：
   - 产品定位（目标人群、使用场景）
   - 核心功能设计（针对痛点的解决方案）
   - 定价策略建议
   - 差异化竞争策略

2. 核心卖点清单（TOP 5）：
   - 卖点描述
   - 支撑证据（来自消费者反馈）
   - 目标人群匹配

3. 产品迭代建议：
   - 优先级排序
   - 实施建议
```

## 3. 虚拟消费者构建方法论

### 3.1 方法论概述

虚拟消费者构建是CottonMind系统的核心创新，其目标是从真实用户数据中提炼出具有可验证决策逻辑的虚拟消费者模型，用于模拟新品概念的市场反馈。

**核心思想**：将消费者决策过程建模为"画像特征 + 决策规则 + 情感表达"的三维模型，通过机器学习和知识工程相结合的方式实现。

### 3.2 技术路线

```
┌─────────────────────────────────────────────────────────────────────┐
│                        虚拟消费者构建流程                            │
├─────────────────────────────────────────────────────────────────────┤

│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                  │
│  │ 用户评论采集 │  │ 行业报告整理│  │ 品牌知识提取│                  │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘                  │
│         │                │                │                         │

│                             ▼                                       │
│  阶段4: 决策逻辑建模                                                 │
│  ┌─────────────────────────────────────────────┐                   │
│  │ 基于评论情感分析的决策规则提取                 │                   │

│  ┌─────────────────────────────────────────────┐                   │
│  │ 输入新品概念 → 执行决策逻辑 → 生成反馈评分     │                   │
│  └─────────────────────────────────────────────┘                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.3 阶段一：数据准备

#### 3.3.1 数据采集

| 数据源 | 采集方式 | 数据类型 | 数据量 |
|--------|---------|---------|--------|
| 电商平台评论 | API接入/爬虫 | 文本 | 10万+ |
| 小红书笔记 | API接入/爬虫 | 文本+图片 | 5万+ |
| 行业报告 | 人工整理 | PDF/文档 | 100+ |
| 品牌知识 | 内部文档 | 结构化数据 | 10+ |

#### 3.3.2 数据预处理

```
原始数据 → 清洗 → 分词 → 去停用词 → 情感标注 → 特征提取
```

#### 3.6.2 权重学习算法

**基于情感分析的权重提取**：

1. 统计每类消费者评论中各产品特征的提及频率
2. 分析提及该特征时的情感倾向（正面/负面）
3. 计算特征重要性权重：`权重 = 提及频率 × 情感强度`

**数学公式**：

```
W_i = (P_positive_i - P_negative_i) × F_i

其中：
- W_i：特征i的权重
- P_positive_i：提及特征i时的正面情感概率
- P_negative_i：提及特征i时的负面情感概率
- F_i：特征i的提及频率
```

#### 3.6.4 评分计算流程

```python
# 评分计算流程示意
def calculate_purchase_intent(product_concept, decision_rules):
    total_score = 0
    total_weight = 0
    
    for rule in decision_rules['rules']:
        feature = rule['feature']
        weight = rule['weight']
        
        # 匹配产品概念中的特征
        matched_condition = find_matching_condition(product_concept, rule['conditions'])
        feature_score = matched_condition['score']
        
        # 加权计算
        total_score += feature_score * weight
        total_weight += weight
    
    # 归一化到1-10分
    final_score = round(total_score / total_weight, 1)
    return final_score
```

### 6.4 飞书工作流设计

#### 6.4.1 虚拟消费者测试流程

```
触发条件：产品团队在飞书多维表格中创建新品概念

步骤1：调用Insight Agent
   - 输入：新品概念、品牌信息
   - 输出：消费者洞察报告、虚拟消费者测试反馈

步骤2：自动记录结果到多维表格
   - 测试反馈数据写入
   - 消费者洞察报告附件上传

步骤3：调用Product Agent
   - 输入：新品概念、测试反馈
   - 输出：优化后产品方案、核心卖点

步骤4：自动记录结果到多维表格
   - 产品方案数据写入
   - 核心卖点清单附件上传

步骤5：调用Marketing Agent
   - 输入：产品方案、核心卖点
   - 输出：营销内容

步骤6：自动记录结果到多维表格
   - 营销内容附件上传

步骤7：调用Growth Agent
   - 输入：测试结果、产品方案
   - 输出：增长分析报告、迭代建议

步骤8：自动记录结果到多维表格
   - 增长分析报告附件上传

步骤9：飞书机器人通知
   - 通知产品团队测试完成
   - 发送结果摘要
```

#### 6.4.2 迭代优化流程

```
触发条件：产品团队在多维表格中更新产品方案

步骤1：检测产品方案变更
   - 对比新版本与旧版本的差异

步骤2：调用Insight Agent重新测试
   - 使用更新后的产品方案
   - 生成新的测试反馈

步骤3：对比分析
   - 分析改进效果
   - 识别新的问题

步骤4：更新多维表格
   - 记录迭代历史
   - 更新测试结果

步骤5：飞书机器人通知
   - 通知迭代完成
   - 发送改进对比报告
```

|---------|---------|------|
| 大语言模型 | 飞书AI模型 | 飞书生态原生支持 |
| 向量数据库 | 飞书知识库内置 | 飞书生态原生支持 |

### B. 关键设计决策

1. **虚拟消费者引擎优先**：将虚拟消费者模拟作为核心创新点，区别于普通的Multi-Agent方案

2. **飞书生态原生**：最大化利用飞书AI能力，减少自定义开发，提升落地效率

3. **Dify早期验证**：使用Dify快速验证工作流程，确保方案可行性

4. **数据驱动决策**：所有虚拟测试基于真实数据，确保结果可信度

5. **迭代闭环设计**：支持产品-测试-优化的持续迭代，符合企业实际工作流程
