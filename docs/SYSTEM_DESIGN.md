### 1.2 架构设计原则

1. **核心引擎优先**：虚拟消费者引擎、适口性预测模型、功效验证模型是整个系统的核心，所有Agent围绕其展开工作

2. **飞书生态原生**：系统设计充分考虑飞书AI生态的能力边界，最大化利用飞书原生能力

3. **模块化设计**：每个Agent独立封装，通过标准化接口协作

4. **数据驱动**：所有决策基于真实数据，确保虚拟测试的可信度

5. **迭代闭环**：支持产品-测试-优化的持续迭代循环

6. **功效验证加速**：将传统1.5-2年的功效验证周期缩短至数小时

## 2. Agent设计

### 2.1 Insight Agent（核心）

#### 身份

**AI市场研究专家 + 虚拟宠主生成器 + 适口性预测器 + 功效验证分析师**

#### 核心职责

作为系统的核心引擎，负责：
- 从真实数据中提炼宠主消费者洞察（成分关注、功能性需求、钙磷比关注度）
- 构建具有决策逻辑的虚拟宠主模型（关注配料来源、钙磷比、添加剂、功能性）
- 模拟宠主对新品概念的反馈
- 基于配方数据预测宠物适口性
- 进行营养成分分析（钙磷比评估、功效验证）

#### 输入

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 品牌信息 | 文本 | 飞书知识库 | 品牌定位、品牌调性、产品矩阵 |
| 产品品类 | 文本 | 用户输入 | 如：冻干零食、磨牙棒、罐头等 |
| 用户评论数据 | 结构化数据 | 数据存储层 | 电商平台、社交媒体评论 |
| 行业报告 | 文档 | 飞书知识库 | 行业趋势、市场规模、竞品分析 |
| 配方数据 | 结构化数据 | 数据存储层 | 原料组合、营养成分、钙磷比、历史反馈 |
| 宠物营养知识 | 文档 | 飞书知识库 | 宠物营养学知识、原料特性、营养标准、钙磷比标准 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 消费者洞察报告 | 结构化报告 | 痛点、需求、偏好分析（成分、功能性、钙磷比） |
| 虚拟宠主模型 | JSON对象 | 包含画像特征和决策逻辑（关注钙磷比、功能性） |
| 测试反馈结果 | 结构化数据 | 不同宠主群体的反馈评分 |
| 适口性预测 | 结构化数据 | 适口性风险、预测评分、优化建议 |
| 功效验证报告 | 结构化数据 | 钙磷比评估、功效预测（美毛/护肠/洁齿）、风险预警 |

#### 工具

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| RAG知识检索 | 从知识库中检索相关信息 | 分析用户评论、行业报告、宠物营养知识 |
| 数据统计分析 | 对评论数据进行统计分析 | 提炼消费痛点和趋势（成分、功能性、钙磷比） |
| 画像聚类 | 对消费者进行群体聚类 | 生成宠主细分群体（Z世代成分党、科学养宠家长等） |
| 决策树生成 | 生成消费者购买决策逻辑 | 构建虚拟宠主决策模型（成分>功能性>钙磷比>品牌>价格） |
| 适口性预测 | 基于配方数据预测适口性 | 评估新品配方的适口性风险 |
| 营养成分分析 | 分析钙磷比、营养配比 | 评估配方营养合理性 |
| 功效验证 | 预测功能性成分效果 | 评估美毛/护肠/洁齿等功效可行性 |

#### 知识库

| 知识库 | 内容 | 更新频率 |
|--------|------|----------|
| 用户评论库 | 电商平台、社交媒体用户评论（含成分、功能性、钙磷比讨论） | 实时 |
| 行业报告库 | 行业分析报告、市场研究报告（功能性零食趋势） | 月度 |
| 品牌知识库 | 品牌历史、产品信息、品牌调性 | 季度 |
| 宠主画像库 | 消费者细分、人群特征（成分党、科学养宠家长等） | 季度 |
| 配方数据库 | 历史配方数据、原料组合、营养成分、钙磷比、适口性反馈 | 月度 |
| 宠物营养库 | 宠物营养学知识、原料特性、营养标准、钙磷比标准 | 季度 |
| 功效验证库 | 功能性成分效果数据、钙磷比历史验证数据 | 月度 |

#### Prompt设计

**角色设定**：
```
你是一名资深宠物食品市场研究专家，专注于宠物健康零食行业。
你拥有丰富的市场研究经验，擅长从海量数据中提炼洞察，并能够构建逼真的虚拟宠主模型。
你还具备宠物营养学知识，能够基于配方数据预测宠物适口性和功效（钙磷比、美毛、护肠、洁齿）。
你的目标是帮助产品团队在新品上市前预测市场反应、宠物适口性和功效可行性。
```

**核心指令**：
```
基于以下数据，完成消费者洞察分析、虚拟宠主模型生成和三维度验证：

1. 用户评论数据：{{reviews_data}}
2. 行业报告：{{industry_reports}}
3. 品牌信息：{{brand_info}}
4. 配方数据：{{recipe_data}}（含钙磷比、营养成分）
5. 宠物营养知识：{{nutrition_knowledge}}

请输出：
1. 消费者洞察报告：
   - 核心消费痛点（TOP 10）：成分透明度、功能性、钙磷比、添加剂等
   - 消费者需求趋势：美毛、护肠、洁齿等功能性需求增长
   - 价格敏感度分析
   - 购买决策因素排序：成分透明度 > 功能性 > 钙磷比 > 品牌 > 价格

2. 虚拟宠主模型（至少5个细分群体）：
   - 群体名称（如：Z世代成分党、科学养宠家长、多宠家庭主理人等）
   - 核心特征（年龄、地域、消费能力、养宠情况等）
   - 购买决策逻辑（决策树，含钙磷比、功能性权重）
   - 产品偏好权重

3. 测试反馈模拟：
   - 针对新品概念{{product_concept}}，模拟各群体的反馈
   - 购买意愿评分（1-10分）
   - 主要顾虑点（成分、钙磷比、功能性）
   - 改进建议

4. 适口性预测：
   - 适口性风险等级（高/中/低）
   - 预测适口性评分（1-10分）
   - 潜在问题分析
   - 配方优化建议

5. 功效验证分析：
   - 营养成分评估（钙磷比、蛋白质含量等）
   - 功效预测（美毛/护肠/洁齿效果评估）
   - 钙磷比合理性评估
   - 风险预警
   - 配方优化建议（钙磷比调整、功能性增强）
```

### 2.2 Product Agent

#### 身份

**宠物食品研发助手 + 功效优化专家**

#### 核心职责

根据虚拟宠主的测试反馈、适口性预测和功效验证结果，优化产品方案，提炼核心卖点（突出功能性），调整配方方向（钙磷比优化、功能性增强）。

#### 输入

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 新品概念 | 文本 | 用户输入 | 产品初步设计方案 |
| 虚拟测试反馈 | 结构化数据 | Insight Agent | 各宠主群体的反馈 |
| 适口性预测 | 结构化数据 | Insight Agent | 适口性风险、优化建议 |
| 功效验证报告 | 结构化数据 | Insight Agent | 钙磷比评估、功效预测、风险预警 |
| 消费者洞察 | 报告 | Insight Agent | 消费痛点、需求趋势（成分、功能性、钙磷比） |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 优化后产品方案 | 结构化文档 | 产品定位、配方方向、功能设计（美毛/护肠/洁齿）、定价策略 |
| 配方优化建议 | 结构化文档 | 原料调整建议、钙磷比优化方向、功能性增强方案 |
| 产品定位报告 | 文本报告 | 目标人群、差异化定位（功能性、成分透明度）、竞争优势 |
| 核心卖点清单 | 列表 | 产品核心价值主张（突出功能性、钙磷比、成分透明度） |

#### 工具

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| 竞品分析 | 分析竞品产品特征、配方、功能性卖点 | 制定差异化策略 |
| 定价策略分析 | 分析价格区间和敏感度 | 制定定价方案 |
| 卖点提炼 | 从产品特征中提炼核心卖点（功能性、钙磷比、成分） | 生成营销素材 |
| 配方优化 | 根据适口性预测和功效验证调整配方 | 优化原料组合、钙磷比调整 |
| 功效增强 | 根据功效验证结果增强功能性成分 | 美毛/护肠/洁齿功效优化 |

#### 知识库

| 知识库 | 内容 | 更新频率 |
|--------|------|----------|
| 竞品数据库 | 竞品产品信息、配方、功能性卖点、定价、评价 | 月度 |
| 产品知识库 | 品牌产品历史、配方数据、技术参数、钙磷比数据 | 季度 |
| 定价策略库 | 行业定价策略、价格区间 | 季度 |
| 配方数据库 | 历史配方数据、原料组合、营养成分、钙磷比、适口性反馈 | 月度 |
| 功效验证库 | 功能性成分效果数据、钙磷比标准、营养标准 | 月度 |

#### Prompt设计

**角色设定**：
```
你是一名资深宠物食品产品经理，专注于宠物健康零食行业。
你擅长根据宠主反馈、适口性预测和功效验证结果优化产品方案。
你精通宠物营养学，能够平衡宠主需求、钙磷比合理性、功能性效果、技术可行性和商业价值。
```

**核心指令**：
```
基于以下信息，优化产品方案并输出产品定位和核心卖点（突出功能性）：

1. 原始新品概念：{{product_concept}}
2. 虚拟宠主测试反馈：{{test_feedback}}
3. 适口性预测：{{palatability_prediction}}
4. 功效验证报告：{{efficacy_report}}（钙磷比评估、功效预测）
5. 消费者洞察报告：{{insight_report}}

请输出：
1. 优化后的产品方案：
   - 产品定位（目标人群、使用场景）
   - 配方方向优化（针对适口性、钙磷比和宠主需求的解决方案）
   - 核心功能设计（美毛/护肠/洁齿等功能性）
   - 定价策略建议
   - 差异化竞争策略（成分透明度、功能性、钙磷比）

2. 配方优化建议：
   - 原料调整建议
   - 钙磷比优化方向（确保在标准范围1.1-1.4内）
   - 功能性增强方案（美毛/护肠/洁齿）
   - 适口性改进方案

3. 核心卖点清单（TOP 5）：
   - 卖点描述（突出功能性、成分透明度、钙磷比）
   - 支撑证据（来自消费者反馈和功效验证）
   - 目标人群匹配

4. 产品迭代建议：
   - 优先级排序（钙磷比调整优先、功能性增强次之）
   - 实施建议
```

### 2.3 Marketing Agent

#### 身份

**宠物品牌内容增长助手 + 功能性卖点营销专家**

#### 核心职责

根据产品方案生成多平台营销内容，突出产品的功能性卖点（美毛、护肠、洁齿）、成分透明度和钙磷比合理性，包括小红书种草内容、抖音短视频脚本、电商详情页、广告文案等。

#### 输入

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 产品方案 | 结构化文档 | Product Agent | 产品定位、配方、功能（美毛/护肠/洁齿）、卖点 |
| 核心卖点 | 列表 | Product Agent | TOP 5核心卖点（突出功能性、成分透明度、钙磷比） |
| 目标人群 | 描述 | Product Agent | 目标宠主画像（Z世代成分党、科学养宠家长等） |
| 功效验证报告 | 结构化数据 | Insight Agent | 钙磷比评估、功效预测数据 |

#### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 小红书文案 | 文本 | 适合小红书平台的宠物种草文案（突出功能性、成分透明度） |
| 短视频脚本 | 结构化脚本 | 抖音/快手短视频分镜脚本（展示功能性效果） |
| 电商详情页 | 结构化内容 | 电商平台产品详情描述（含钙磷比信息、功效说明） |
| 广告文案 | 文本 | 适合各平台的广告文案（突出保健品级功效） |
| 传播策略 | 文本报告 | 多平台传播建议（针对成分党、科学养宠家长） |

#### 工具

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| 文案生成 | 根据产品卖点生成营销文案（突出功能性、钙磷比） | 各平台内容创作 |

## 3. 虚拟宠主构建方法论

### 3.1 方法论概述

虚拟宠主构建是PetMind系统的核心创新，其目标是从真实用户数据中提炼出具有可验证决策逻辑的虚拟宠主模型，用于模拟新品概念的市场反馈。

**核心思想**：将宠主决策过程建模为"画像特征 + 决策规则 + 情感表达"的三维模型，同时整合"适口性预测 + 功效验证"两大核心能力，通过机器学习和知识工程相结合的方式实现。

宠物健康零食行业的特殊之处在于：
1. **购买者≠使用者**：购买决策由宠主完成，但实际使用者是宠物
2. **功能性需求突出**：宠主关注美毛、护肠、洁齿等保健品级功效
3. **科学养宠意识强**：Z世代宠主深度追问配料来源、钙磷比、添加剂

因此，我们需要构建三维度验证体系：
1. **虚拟宠主模型**：模拟宠物主人的购买决策过程（关注成分、功能性、钙磷比）
2. **适口性预测模型**：基于配方数据预测宠物对产品的接受程度
3. **功效验证模型**：基于宠物营养学知识，评估钙磷比、功能性成分效果

- **特征提取**：LLM提取产品相关特征（配方、适口性、营养成分、钙磷比、功能性、价格、包装等）
- **配方解析**：从配方数据中提取原料组合、营养成分、钙磷比等结构化信息
- **营养分析**：计算钙磷比、蛋白质含量等关键营养指标

### 3.4 阶段二：消费者聚类

#### 3.4.1 聚类策略

采用**语义相似度聚类**方法，将具有相似观点和偏好的宠主聚为一类。

#### 3.4.2 技术实现

```python
# 聚类流程示意
embeddings = model.encode(reviews_text)  # 文本向量化
clusters = KMeans(n_clusters=5).fit(embeddings)  # K-Means聚类
cluster_labels = clusters.labels_  # 获取每个评论的类别标签
```

#### 3.4.3 聚类参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 聚类数量 | 5-10 | 根据数据量和多样性调整，识别不同宠主群体（如Z世代成分党、多宠家庭主理人等） |
| 向量维度 | 768/1024 | 取决于Embedding模型 |
| 距离度量 | 余弦相似度 | 适合文本语义相似度计算 |

#### 3.4.4 聚类结果验证

- **轮廓系数**：评估聚类质量，目标 > 0.5
- **人工验证**：抽样检查每个聚类的一致性
- **业务验证**：聚类结果是否符合业务认知的宠主群体（如Z世代成分党、多宠家庭主理人等）

#### 3.6.2 权重学习算法

**基于情感分析的权重提取**：

1. 统计每类宠主评论中各产品特征的提及频率
2. 分析提及该特征时的情感倾向（正面/负面）
3. 计算特征重要性权重：`权重 = 提及频率 × 情感强度`

**数学公式**：

```
W_i = (P_positive_i - P_negative_i) × F_i

其中：
- W_i：特征i的权重
- P_positive_i：提及特征i时的正面情感概率
- P_negative_i：提及特征i时的负面情感概率
- F_i：特征i的提及频率
```

#### 3.6.5 评分计算流程

```python
# 评分计算流程示意
def calculate_purchase_intent(product_concept, decision_rules):
    total_score = 0
    total_weight = 0
    
    for rule in decision_rules['rules']:
        feature = rule['feature']
        weight = rule['weight']
        
        # 匹配产品概念中的特征
        matched_condition = find_matching_condition(product_concept, rule['conditions'])
        feature_score = matched_condition['score']
        
        # 加权计算
        total_score += feature_score * weight
        total_weight += weight
    
    # 归一化到1-10分
    final_score = round(total_score / total_weight, 1)
    return final_score
```

#### 3.7.2 适口性预测算法

1. 收集历史配方数据和对应的适口性反馈

3. 训练机器学习模型（如随机森林、梯度提升树）

**特征工程**：

| 特征类型 | 具体特征 |
|---------|---------|
| 原料特征 | 主要原料种类、原料比例、原料新鲜度 |

| 调味特征 | 是否添加诱食剂、风味类型 |

**数学公式**：

```
P_palatability = f(ingredients, nutrition, processing, flavor)

其中：
- P_palatability：适口性预测概率
- ingredients：原料特征向量
- nutrition：营养特征向量
- processing：加工特征向量
- flavor：调味特征向量
- f：机器学习预测函数
```

### 3.10 模型验证与迭代

#### 3.10.1 验证指标

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| 预测准确率 | 虚拟测试结果与真实市场反馈的一致性 | ≥ 80% |
| 适口性预测准确率 | 适口性预测与真实宠物测试结果的一致性 | ≥ 85% |
| 反馈多样性 | 不同群体反馈的差异化程度 | ≥ 0.7 |
| 反馈可信度 | 人工评估反馈的合理性和真实性 | ≥ 4.5/5 |
| 计算效率 | 单次测试的平均耗时 | ≤ 30秒 |

#### 3.10.2 迭代优化

```
虚拟测试 + 适口性预测 → 真实市场验证 + 宠物测试 → 偏差分析 → 模型调整 → 重新测试

            风险评估      真实用户反馈和    反馈的差异        优化预测模型
                        宠物测试结果
```

### 6.4 飞书工作流设计

#### 6.4.1 虚拟宠主测试流程

```
触发条件：产品团队在飞书多维表格中创建新品概念（含配方信息）

步骤1：调用Insight Agent
   - 输入：新品概念、配方信息、品牌信息
   - 输出：消费者洞察报告、虚拟宠主测试反馈、适口性预测

步骤2：自动记录结果到多维表格
   - 测试反馈数据写入
   - 适口性预测数据写入
   - 消费者洞察报告附件上传

步骤3：调用Product Agent
   - 输入：新品概念、测试反馈、适口性预测
   - 输出：优化后产品方案、配方优化建议、核心卖点

步骤4：自动记录结果到多维表格
   - 产品方案数据写入
   - 配方优化建议数据写入
   - 核心卖点清单附件上传

步骤5：调用Marketing Agent
   - 输入：产品方案、核心卖点
   - 输出：营销内容

步骤6：自动记录结果到多维表格
   - 营销内容附件上传

步骤7：调用Growth Agent
   - 输入：测试结果、适口性预测、产品方案
   - 输出：增长分析报告、产品迭代建议、市场机会分析

步骤8：自动记录结果到多维表格
   - 增长分析报告附件上传

步骤9：飞书机器人通知
   - 通知产品团队测试完成
   - 发送结果摘要（含适口性预测）
```

#### 6.4.2 迭代优化流程

```
触发条件：产品团队在多维表格中更新产品方案或配方

步骤1：检测产品方案或配方变更
   - 对比新版本与旧版本的差异

步骤2：调用Insight Agent重新测试
   - 使用更新后的产品方案和配方
   - 生成新的测试反馈和适口性预测

步骤3：对比分析
   - 分析改进效果
   - 分析适口性变化
   - 识别新的问题

步骤4：更新多维表格
   - 记录迭代历史
   - 更新测试结果
   - 更新适口性预测

步骤5：飞书机器人通知
   - 通知迭代完成
   - 发送改进对比报告（含适口性变化）
```

### B. 关键设计决策

1. **双维度虚拟测试优先**：将虚拟宠主测试和适口性预测作为核心创新点，区别于普通的Multi-Agent方案

2. **飞书生态原生**：最大化利用飞书AI能力，减少自定义开发，提升落地效率

3. **Dify早期验证**：使用Dify快速验证工作流程，确保方案可行性

4. **数据驱动决策**：所有虚拟测试基于真实数据，确保结果可信度

5. **迭代闭环设计**：支持产品-测试-优化的持续迭代，符合企业实际工作流程

6. **宠物食品行业特性**：针对宠物食品行业"购买者≠使用者"的特殊问题，同时构建虚拟宠主模型和适口性预测模型
